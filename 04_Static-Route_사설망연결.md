# 04. Static Route — 본사·지사 사설망 연결

## 배경

- ISP Router의 라우팅 테이블에는 사설 네트워크 정보가 없다.
- GIT-A/B 라우팅 테이블에도 상대방 사설망 정보가 없다.
- GRE 터널을 next-hop으로 하는 정적 경로를 추가하여 사설망을 연결한다.

---

## 설정

### GIT-A (R1)

```
ip route 192.168.20.0 255.255.255.0 172.16.10.2
```

```
! -------- 또는 (outgoing interface 사용) --------
ip route 192.168.20.0 255.255.255.0 tunnel 12
```

### GIT-B (R2)

```
ip route 192.168.10.0 255.255.255.0 172.16.10.1
```

```
! -------- 또는 (outgoing interface 사용) --------
ip route 192.168.10.0 255.255.255.0 tunnel 12
```

---

## 정보 확인

```
GIT-A# show ip route
      172.16.0.0/24 is subnetted, 1 subnets
C        172.16.10.0 is directly connected, Tunnel12
C    192.168.10.0/24 is directly connected, FastEthernet0/0   <-- GIT-A 내부 사설망
S    192.168.20.0/24 [1/0] via 172.16.10.2                    <-- GIT-B 내부 사설망

PC1# ping 192.168.20.2
```

```
GIT-B# show ip route
      172.16.0.0/24 is subnetted, 1 subnets
C        172.16.10.0 is directly connected, Tunnel12
S    192.168.10.0/24 [1/0] via 172.16.10.1                    <-- GIT-A 내부 사설망
C    192.168.20.0/24 is directly connected, FastEthernet0/1   <-- GIT-B 내부 사설망

PC2# ping 192.168.10.1
```

```
# PC1 <----- PC2
========================================================

========================================================
```
